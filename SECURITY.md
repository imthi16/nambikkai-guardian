# Security Policy

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability
reporting for this repository. Do not include real credentials, private documents, personal data,
or production prompts in a report; use minimal sanitized examples.

Include the affected revision, impact, safe reproduction, and any known tenant-isolation or
prompt-injection implications. Maintainers should acknowledge the report before discussing a
public disclosure timeline.

## Supported versions

Until the first stable release, security fixes are applied to the latest revision of `main` only.
